# Explain Code

Explain the current file or the code described. Tailor the explanation to the user's level.

Structure:
1. **What it does** — one sentence purpose statement
2. **How it works** — data flow, key decisions, non-obvious logic (with line refs)
3. **Why it's built this way** — trade-offs made, alternatives rejected
4. **Gotchas** — things that will surprise a new contributor
5. **Entry points** — where to start reading / where to make changes

Be specific. Reference actual function names, line numbers, variable names from the code.
Do not pad with generic advice.
